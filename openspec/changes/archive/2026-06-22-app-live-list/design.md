## Context

首页 `HomeLiveSection` 已实现横向直播卡片预览，Mock 数据仅 2 条，「更多」按钮 Toast 占位。设计稿「全部房间」页为独立全屏页，展示完整纵向列表，卡片含品类标签与连麦状态，与首页横向卡片信息密度不同。

现有路由模式：`/im/list`、`/im/chat/:expertId` 为 App 框架外的独立全屏页；全部直播页采用相同模式，路由 `/live/list`，使用 `van-nav-bar` 返回。

## Goals / Non-Goals

**Goals:**

- 实现 `/live/list` 全部直播页，UI 对齐设计稿
- 首页「更多」跳转至该页，返回可回到首页 Tab
- 扩展 Mock 数据与类型，支持 status、多品类展示
- 卡片点击 Toast 占位，不进入真实直播间

**Non-Goals:**

- 真实直播间播放、连麦、支付流程
- 直播列表筛选、搜索、分页
- 下拉刷新（首版可不加，与 IM 列表模式一致时可后续补充）

## Decisions

### 1. 路由：`/live/list` 独立全屏页

**选择**: 与 `/im/list` 一致，注册在 `/app` 子路由之外，`meta: { requiresAuth: true }`。

**理由**: 全屏列表需隐藏底部 Tab 栏；与 IM 列表架构一致，PM 演示流程清晰。

**替代方案**: 作为 `/app` 子路由 — 会保留 Tab 栏，不符合设计稿全屏体验。

### 2. 数据复用：`getLiveRooms()` 共享 Mock

**选择**: 全部直播页与首页共用 `getLiveRooms()`，Mock 数据扩充至 6+ 条。

**理由**: 演示项目无需独立 API；首页预览与列表页数据一致更合理。

**替代方案**: 新增 `getAllLiveRooms()` — 过度设计，首版不必要。

### 3. 类型扩展：`LiveRoomItem.status`

**选择**: 新增 `status: '空闲中' | '咨询中'`，Mock 中混合两种状态。

**理由**: 设计稿明确展示状态标识与不同文字颜色；首页横向卡片可不展示 status，字段可选或首页忽略。

### 4. 品类标签颜色映射

**选择**: 组件内按 `category` 字符串映射背景色：

| 品类 | 背景色 |
|------|--------|
| 塔罗 | 紫色 `rgba(155, 111, 208, 0.25)` |
| 国学 | 橙棕 `rgba(200, 140, 80, 0.25)` |
| 占星 | 蓝色 `rgba(80, 140, 220, 0.25)` |

**理由**: 设计稿三种标签颜色不同；演示阶段硬编码映射即可。

### 5. 页面布局

```
┌─────────────────────────────┐
│  ←  全部房间                 │  van-nav-bar
├─────────────────────────────┤
│ ┌──────┬──────────────────┐  │
│ │ 封面 │ 标题              │  │  LiveRoomListItem × N
│ │      │ [塔罗] 📊 空闲中  │  │
│ │      │ 👤 暖暖塔罗咨询师  │  │
│ └──────┴──────────────────┘  │
│         ...                   │
└─────────────────────────────┘
```

卡片：横向 flex，左 200px 封面 + 右 flex 信息区；列表 `padding: 24px`，卡片间距 20px，圆角 16px，背景 `$card-bg`。

### 6. 导航流程

- 首页 `handleLiveMore()` → `router.push('/live/list')`
- 全部直播页返回 → `router.back()` 或 `router.push('/app/home')`

## Risks / Trade-offs

- **[Risk] Mock 数据与首页预览条数不一致** → 扩充 Mock 至 6 条，首页仍只展示前 2 条（横向滚动区域按全量或 slice 展示，设计稿首页仅 2 条，列表页展示全部）
- **[Risk] status 字段首页未使用** → 类型统一扩展，首页组件忽略该字段，无 breaking change
- **[Risk] 返回栈** → 使用 `router.back()`，从首页进入可正确返回
